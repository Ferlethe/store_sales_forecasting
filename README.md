# Store Sales Forecasting

Time series forecasting model to predict store sales using data from Corporación Favorita. Covers exploratory data analysis, feature engineering with external data (oil prices, holidays, transactions), and comparison of forecasting models.

---

## Objective

Build a machine learning model capable of predicting daily sales for thousands of product families across 54 stores, using historical sales data combined with external factors such as oil prices, holidays, and transaction volume.

This project follows a complete end-to-end data science pipeline, from raw data to model evaluation.

---

## Dataset

⚠️ **Note:** The raw dataset files (especially `train.csv`) are large (100MB+) and are **not included in this repository** due to GitHub's file size limits. They must be downloaded externally before running the notebooks.

Download the dataset at [Store Sales - Time Series Forecasting | Kaggle](https://www.kaggle.com/competitions/store-sales-time-series-forecasting/data)

Place all six CSV files in `data/raw/`:
```
data/raw/
├── train.csv
├── test.csv
├── stores.csv
├── holidays_events.csv
├── oil.csv
└── transactions.csv
```

Processed datasets generated by the notebooks (`train_cleaned.csv`, `train_featured.csv`) are also excluded from version control, as they can be regenerated by running the notebooks in order.

## Project Overview

The workflow is divided into five main stages:

* **Data Loading**
  Loading and inspecting all six datasets (train, test, stores, holidays, oil, transactions) and understanding how they relate to each other

* **Data Cleaning**
  Converting date columns, treating missing oil prices with linear interpolation, merging all datasets into a single table, and engineering a national holiday flag

* **Exploratory Data Analysis (EDA)**
  Analyzing sales trends, weekly and yearly seasonality, top product categories, and the relationship between oil prices and sales

* **Feature Engineering**
  Creating time-based features, lag features (7/14/28-day), and rolling mean features per store/family group

* **Modeling**
  Training and comparing Random Forest, XGBoost, and LightGBM, with manual hyperparameter tuning

---

## Key Insights

* Sales show a clear **upward trend** from 2013 to 2017
* **January 1st (New Year)** consistently shows near-zero sales across all years
* **Weekends** have significantly higher sales than weekdays
* **GROCERY I** and **BEVERAGES** are by far the top-selling product categories
* Oil price shows an **inverse relationship** with sales, likely tied to Ecuador's oil-dependent economy
* **Lag and rolling mean features** were the strongest predictors in the final model — recent sales history drives forecasting accuracy more than any other variable

## Model Performance

| Model | RMSLE |
|---|---|
| Random Forest | 0.5798 |
| XGBoost | 1.1936 |
| LightGBM | 1.0503 |

* **Best Model:** Random Forest Regressor (tuned)
* **Final Validation RMSLE:** 0.5715
* Validation was done using a **time-based split** (not random), respecting chronological order to avoid data leakage
